🏗️ 01 — QU'EST-CE QUE BUILDRS
Structure groupe
Buildrs Group          ← Société mère ├── Buildrs Lab        ← Le laboratoire : Micro-SaaS créés par Alfred en live └── Buildrs Academy    ← La formation : cohortes, guides, accompagnement
Buildrs est un écosystème SaaS IA fondé par Alfred Orsini. Positionnement : "Le Laboratoire des Builders SaaS IA"
Domaine principal : buildrs.fr Personal brand : alfred-orsini.com
Mission
Démocratiser la création de Micro-SaaS en accompagnant des entrepreneurs, salariés, étudiants et créateurs à construire leurs premiers actifs logiciels grâce au VibeCoding et à l'IA — sans compétences techniques avancées.
Ce que Buildrs résout
* La confusion massive autour de l'IA comme levier business
* La croyance fausse que créer un SaaS nécessite de savoir coder
* L'absence de méthode terrain crédible sur le marché francophone
* Le manque de framework complet : idée → validation → build → MRR → exit
Concept clé : Orchestrateur d'IA
Un utilisateur Buildrs n'est pas un développeur. C'est un chef d'orchestre : il dirige plusieurs agents IA autonomes pour construire des actifs logiciels réels. Ne pas savoir coder est un avantage — pas un handicap.
Les 3 stratégies enseignées
1. Copier & Adapter — SaaS étranger qui cartonne → adapter au marché français
2. Résoudre un Problème — Douleur observée → valider → construire la solution
3. Commande Privée — Build sur commande pour une entreprise (cash immédiat)
Concept Micro-SaaS = Agent IA
Un Micro-SaaS Buildrs = un ou plusieurs agents IA autonomes encapsulés dans une interface utilisateur qui résout UN problème précis pour UNE cible précise.

⚡ 04 — LES 5 GÉNÉRATEURS IA
Ordre de build prioritaire
🥇 1 — G4 AI Business Matcher   → PREMIER (viral, top of funnel, audience large) 🥈 2 — G2 Idea Validator         → DEUXIÈME (forte qualification, intent élevé) 🥉 3 — G1 Clone Finder           → TROISIÈME (viral LinkedIn/Twitter)    4 — G3 Problem to SaaS        → Audience engagée requise    5 — G5 Agent SaaS Builder     → Dashboard members uniquement
Architecture commune de chaque générateur
1. Input    : formulaire simple — max 3 champs — mobile-first 2. Process  : appel lib/ai.ts → Edge Function Supabase → Anthropic API 3. Output   : JSON structuré → UI formatée avec sections claires 4. Gate     : 1 use gratuit → modal inscription → upsell page offres
G4 — AI Business Matcher ⭐ PRIORITÉ 1
Input   : quiz 5 questions           Q1 — domaine d'expertise / secteur actuel           Q2 — niveau tech (curseur 0=aucun → 5=dev)           Q3 — budget départ estimé (< 500€ / 500-2000€ / 2000€+)           Q4 — objectif (MRR stable / revendre / liberté géo / tout)           Q5 — appétit risque (faible / moyen / élevé)  Output  : {   profile_name: string,          // ex: "Le Builder Stratège"   profile_description: string,   business_type: string,         // "Micro-SaaS" | "App IA" | "SaaS niche"   strategy: "clone|problem|commission",   recommended_niche: string,   first_step: string,            // Action concrète cette semaine   tools: string[3],   estimated_timeline: string,   why_you: string                // Pourquoi ce profil lui correspond }
G2 — Idea Validator ⭐ PRIORITÉ 2
Input   : description idée en texte libre (max 200 chars)  Output  : {   score: number,                 // 0-100   score_label: string,           // "Faible" | "Moyen" | "Fort" | "Excellent"   market_size: string,   competitors: string[],   icp: string,   acquisition_channel: string,   main_risk: string,   verdict: "LANCE" | "PIVOTE" | "OUBLIE",   verdict_reason: string,   next_step: string }
G1 — Clone Finder ⭐ PRIORITÉ 3
Input   : domaine d'intérêt (ex: "fitness", "RH PME")  Output  : {   saas_name: string,   origin_country: string,   why_it_works: string,   fr_opportunity: string,   market_size_fr: string,   competitors_fr: string[],   positioning_angle: string,   mvp_features: string[3],   stack_suggested: string,   estimated_build_time: string }
G3 — Problem to SaaS
Input   : description d'un problème observé  Output  : {   saas_name: string,   tagline: string,   problem_statement: string,   solution: string,   mvp_features: string[3],   pricing_suggestion: string,   target_persona: string,   acquisition_hint: string,   agent_ia_role: string }
G5 — Agent SaaS Builder (dashboard uniquement)
Input   : secteur + problème cible (optionnel)  Output  : {   concept_name: string,   tagline: string,   problem: string,   solution: string,   agent_role: string,   mvp_features: string[5],   tech_stack: string,   monetization: string,   icp_detailed: string,   go_to_market: string }
